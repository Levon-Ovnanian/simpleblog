{\bfseries{How does \mbox{\hyperlink{class_parsedown}{Parsedown}} work?}}

It tries to read Markdown like a human. First, it looks at the lines. It’s interested in how the lines start. This helps it recognise blocks. It knows, for example, that if a line starts with a {\ttfamily -\/} then perhaps it belongs to a list. Once it recognises the blocks, it continues to the content. As it reads, it watches out for special characters. This helps it recognise inline elements (or inlines).

We call this approach \char`\"{}line based\char`\"{}. We believe that \mbox{\hyperlink{class_parsedown}{Parsedown}} is the first Markdown parser to use it. Since the release of \mbox{\hyperlink{class_parsedown}{Parsedown}}, other developers have used the same approach to develop other Markdown parsers in PHP and in other languages.

{\bfseries{Is it compliant with Common\+Mark?}}

It passes most of the Common\+Mark tests. Most of the tests that don\textquotesingle{}t pass deal with cases that are quite uncommon. Still, as Common\+Mark matures, compliance should improve.
